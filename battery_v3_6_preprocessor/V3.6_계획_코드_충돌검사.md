# v3.6 전처리 계획·코드 충돌 검사

## 판정

현재 코드는 v3.4 전체 구현을 기반으로 v3.5 결정과 v3.6 CT 면적합 층화를 순서대로 반영한다. 계획 우선순위는 `v3.6 > v3.5 > v3.4`다.

## 버전 간 충돌과 적용 결정

| 항목 | v3.4 | v3.5 | v3.6 코드 판정 |
|---|---|---|---|
| CT development 이미지 | 축별 100장 표본 | 각 ID의 모든 유효 x/y/z 이미지 | v3.5 전량 사용 적용 |
| CT 축 적격성 | 축별 100장 충족 조건 | 적격성 게이트 제거, 진단만 | v3.5 적용 |
| CT Test | 축·결함비 균형 | 축 우선 후 결함비 균형 | 유지, 면적 구간을 마지막 보조 지표로 추가 |
| CT fold | defect ratio serpentine | 전량 기준 serpentine + 밀도 swap | ID 8개 정원, defect ratio, 밀도 게이트 유지 + 축·면적 목적함수 추가 |
| annotation 밀도 | ±20% 게이트 | ±20%, swap 목표 15% | 그대로 유지 |
| ID 누수 | 금지 | 금지 | 그대로 유지, 면적 균형보다 우선 |
| RGB 160 ID 및 128/16/16 | 적용 | 적용 | 변경 없음 |
| RGB train/val 300장 제한 | 적용 | 적용 | 변경 없음 |
| CT development/Test 300장 제한 | 제거 | 전량 | 전량 유지 |

## v3.6 구현 위치

- `battery_v3_6/ct_area.py`: 면적 구간과 CT 축×면적 feature
- `battery_v3_6/geometry.py`: ROI 기준 유효·클리핑 후 polygon 면적비 계산
- `battery_v3_6/scan.py`: 이미지별 porosity 폴리곤 수·면적합 저장
- `battery_v3_6/selection.py`: 기존 지표와 CT 축·면적을 함께 사용한 ID 단위 배정
- `battery_v3_6/reports.py`: manifest 컬럼과 CT 면적 분포·검수 CSV

## 검증 결과

- Python compile 검사 통과
- 전체 단위·통합 테스트 31건 통과
- CT fold당 8 ID 구조 검사 통과
- CT development 전량 선택 검사 통과
- Test/development 축 균형 검사 통과
- 원본 JSON byte 보존 검사 통과
- detection/segmentation 출력 역검증 통과
- split/fold ID·pixel hash 누수 검사 코드 유지

실제 원본 전체 execute는 대용량 파일 생성과 승인본 확정이 필요한 작업이므로 자동 수행하지 않았다. 첫 실행에서는 dry-run 결과의 경고, ID 배정표, CT 면적 분포를 검토한 뒤 승인해야 한다.
